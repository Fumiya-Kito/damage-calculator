# ポケモンダメージ計算アプリ設計書

## 概要

**ポケモンチャンピオンズ**向けのダメージ計算ツール。
技は1つに固定し、**受け側（防御側）のH×防御ステータス×性格補正の簡易パターン（2×2×2=8通り）を自動生成し、確定発数が切り替わる境界を一目で確認できる**ことが最大の特徴。

### 前提条件
- レベル50固定
- 個体値6V（31オール）固定
- 技は固定で1つのみ選択
- **努力値は0～32振り**（チャンピオンズの仕様。従来の0～252ではない）
- 攻撃側の使用ステータス（A/C）は、選択した技の分類（物理/特殊）から自動判定
- 受け側のパラァータはHと（技の分類に対応する）BまたはDの2つのみ
- 受け側の努力値は一攦0か32のみを想定（ユーザーには選ばせず自動で網羅）

## ダメージ計算式

```
ダメージ = ((レベル × 2 / 5 + 2) × 威力 × 攻撃 / 防御 / 50 + 2) 
         × 乱数(0.85〜1.00) 
         × タイプ一致補正
         × タイプ相性
         × 急所補正
         × その他補正
```

## 主要機能

### コア機能（最優先）
- [ ] 攻撃側設定（固定1パターン）
  - ポケモン選択
  - 技の選択（1つのみ、威力・タイプ・分類（物理/特殊）を保持）
  - 技の分類からAまたはCを自動選択（ユーザーは意識しなくてよい）
  - 努力値（0～32）を対応するステータス（AまたはC）のみ入力
  - 性格補正
- [ ] 防御側パターン自動生成 **（コア）**
  - ポケモンは1体を選択するだけ（ユーザーがステータスを入力する必要はない）
  - パラァータは HのEV(0/32) × 防御ステータスEV(0/32) × 性格補正（無補正/32振っている方の防御ステータスを上げる性格）の3軸
  - 2×2×2 = **8通りを自動で全て生成**（ユーザーは選択しない）
  - 防御ステータスはBとDのどちらかだけ（攻撃側の技の分類で自動決定）
- [ ] ダメージ結果一括表示 **（コア）**
  - 行 = 自動生成された8通りの防御パターン
  - 列 = H EV、防御EV、性格、最小ダメージ、最大ダメージ、割合（%）、確定n発
  - 確定発数が切り替わる境界を視覚的に強調（例: 行の背景色を切り替わり目で分けるなど）

### 追加機能（余裕があれば）
- 0/32以外の任意の努力値も指定できるように拡張
- 持ち物補正（こだわりハチマキ/メガネなど）
- ランク補正（攻撃↑↑など）
- 壁補正
- 天候・フィールド補正
- 特性補正
- パターンの保存/テンプレート化（よく使う振りのプリセット）

### 操作フロー
1. 攻撃側（固定1体）のポケモン・性格・技（1つ）を選択し、対応する努力値（0～32）を入力
2. 防御側のポケモンを選択（ステータス入力は不要）
3. **H EV(0/32) × 防御EV(0/32) × 性格補正の有無 = 8行を自動生成して一括計算**
4. 確定発数が切り替わる境界行が一目ですぐ分かる

## データ型

```typescript
// ポケモン種族値
interface PokemonBase {
  name: string;
  baseStats: {
    hp: number;
    attack: number;
    defense: number;
    specialAttack: number;
    specialDefense: number;
    speed: number;
  };
  types: [PokemonType, PokemonType?];
}

// ポケモンチャンピオンズの努力値（0～32の2値のみを想定）
type EVValue = 0 | 32;

// 性格補正
type Nature = 
  | 'いじっぱり' | 'ようき' | 'ひかえめ' | 'おくびょう' 
  | 'わんぱく' | 'しんちょう' | 'ずぶとい' | 'おだやか'
  | 'まじめ' // がんばりや等の補正なしも
  // ... 他の性格

// 攻撃側（固定1体、技は1つ）
interface Attacker {
  pokemon: PokemonBase;
  nature: Nature;
  offensiveEV: number; // 0～32。使うステータス（A/C）は move.category から自動決定
  move: Move; // 技は1つのみ
}

// 防御側のポケモン（ステータスはユーザー入力不要）
interface DefenderPokemon {
  pokemon: PokemonBase;
}

// 自動生成され〓8パターンの1行分（H EV × 防御EV × 性格補正有無）
interface DefenderStatRow {
  hpEV: EVValue;
  defEV: EVValue; // move.category が physical ならB、special ならD のEV
  natureBoost: boolean; // true = 対応する防御ステータスを上げる性格、false = 無補正
}

// 8行の自動生成（H:2 × 防御:2 × 性格補正:2 = 8通り）
function generateDefenderStatRows(): DefenderStatRow[] {
  const evValues: EVValue[] = [0, 32];
  const rows: DefenderStatRow[] = [];
  for (const hpEV of evValues) {
    for (const defEV of evValues) {
      for (const natureBoost of [false, true]) {
        rows.push({ hpEV, defEV, natureBoost });
      }
    }
  }
  return rows; // 長さ 8 の配列
}

// 技データ
interface Move {
  name: string;
  power: number;
  category: 'physical' | 'special';
  type: PokemonType;
  priority?: number; // 先制技など
}

// タイプ
type PokemonType = 
  | 'ノーマル' | 'ほのお' | 'みず' | 'でんき' | 'くさ' | 'こおり'
  | 'かくとう' | 'どく' | 'じめん' | 'ひこう' | 'エスパー'
  | 'むし' | 'いわ' | 'ゴースト' | 'ドラゴン' | 'あく' | 'はがね' | 'フェアリー';

// ダメージ計算結果（1行分）
interface DamageRow {
  statRow: DefenderStatRow;
  minDamage: number;
  maxDamage: number;
  minPercent: number;  // 割合
  maxPercent: number;
  guaranteedHits: number; // 確定で倒せる最小回数
  rolls: string; // "確定1発" "乱数2発(87.5%)" など
}

// ダメージ計算結果（全体）
interface DamageTable {
  attacker: Attacker;
  defenderPokemon: DefenderPokemon;
  results: DamageRow[]; // 常に8行（自動生成）
}
```
